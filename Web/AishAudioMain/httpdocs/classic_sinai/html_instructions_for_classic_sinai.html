The white box on the page that hold the program descriptions
is a series of nested tables. This document will explain in
detail how the page is to be built to insure the cleanest layout.

The graphic, the description and the links are in a table.
Between each table is a graphic that makes the line. It is not
an hr, because they add extra vertical spacing we don't want.

These tables sit in a larger holding table.
The grey vertical line that runs down the middle of the page
come from the bgcolor of a 2 pixel wide td. On both sides of that
td is a spacer td to maintain the space between the columns.

It is very important that the black descriptive text NOT
extend past the blue program links for each program.
If this happens, the left and right columns will not
line up, meaning the little grey lines that separate the
programs will not line up, and it will look... bad.
Explain this to Aaron, and maybe even show him,
so he knows to write short descriptions.

All this is labelled in the code below.

There is a style sheet that is used for
font colors and sizes only. It must sit
in the same level as this page, or else
change the address in the page header
to point to where it will be.
